# 日麻點數計算助手 - 企劃書

## 一、專案背景與目標

### 1.1 背景分析

日本麻將（Riichi Mahjong）是一種具有高度策略性和複雜計分系統的棋牌遊戲。與線上麻將不同，實體日麻要求玩家能夠快速準確地計算點數，包括：

- **役種判定**：超過30種不同的役種組合
- **番符計算**：複雜的符數與番數對應規則
- **場況修正**：根據親家/子家、本場、連風等調整
- **支付計算**：根據勝利方式分配點數

對於從線上轉向實體的新手而言，計算點數往往成為最大的門檻。

### 1.2 專案目標

開發一個**跨平台網頁應用程式**，讓玩家能夠：

1. 隨時隨地透過手機瀏覽器輸入手牌
2. 自動解析手牌與鳴牌
3. 快速識別役種
4. 準確計算最終得分
5. 支持線上/線下使用

## 二、核心功能規劃

### 2.1 功能模組表

| 功能模組 | 詳細描述 | 優先級 |
|--------|--------|------|
| **手牌解析引擎** | 將使用者輸入的牌型（如：111s 222s...）解析為程式邏輯可運算的資料結構 | P0 |
| **役種與番符判定** | 自動辨識常見役種（立直、斷么、平胡、一盃口等）並依據符數規則計算基礎點 | P0 |
| **點數轉換邏輯** | 考慮場風、自風、親家/子家權重，以及自摸/榮和的點數分配 | P0 |
| **響應式網頁介面** | 利用 Streamlit 打造適配手機螢幕的互動式 UI，方便在牌桌上即時使用 | P1 |
| **族性計算** | 計算平胡、斷么等常見族性 | P2 |
| **計算歷史** | 記錄過去的計算結果 | P3 |

### 2.2 手牌解析引擎

**輸入格式**：
```
123m456p789s11z
```

其中：
- `m` = 萬子 (Man)
- `p` = 筒子 (Pin)
- `s` = 索子 (Sou)
- `z` = 字牌 (Honor)

**解析流程**：
1. 驗證輸入格式
2. 正則表達式提取牌型與花色
3. 轉換為內部資料結構
4. 驗證牌數與合法性

### 2.3 役種判定

**支持的役種**（按番數分類）：

**1番役**：
- 立直 (Riichi)
- 門前清模和 (Menzen Tsumo)
- 平胡 (Pinfu)
- 斷么九 (Tanyao)
- 一盃口 (Iipeiko)
- 一氣通貫 (Sanshoku Doujun)
- 三色同刻 (Sanshoku Doukou)

**2-4番役**：
- 混老頭 (Honroutou)
- 混全帶么九 (Honchantaghan)
- 混一色 (Honisou)
- 三槓子 (Sankantsu)
- 三暗刻 (Sanankou)

**5番役**：
- 清一色 (Chiisou)

**役滿**：
- 大三元 (Daisangen)
- 大四喜 (Daisushi)
- 清老頭 (Chinroutou)
- 字一色 (Tsuisou)
- 綠一色 (Ryuisou)
- 國士無双 (Kokushi)
- 四暗刻 (Suankou)
- 九蓮寶燈 (Chuuren Poutou)

### 2.4 計分系統

**基本概念**：
- **符數 (Fu)**：基礎符 (20) + 各種加符
- **番數 (Han)**：根據役種確定
- **基本點 (Base Points)**：符 × 2^(番+2)，上限1920

**支付計算**：

根據 `番-符` 組合表進行查表：
```
(1, 30) -> 親自摸: 500, 子自摸: 300, 榮和: 1000
(1, 40) -> 親自摸: 700, 子自摸: 400, 榮和: 1300
...
(滿貫) -> 親自摸: 3000, 子自摸: 2000, 榮和: 6000
```
